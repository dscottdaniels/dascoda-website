# Webflow Component Guidance

## Header
- Use a simple horizontal desktop nav
- Use a clean mobile menu
- Keep the CTA visible on desktop
- Avoid oversized navigation treatments

## Section System
- Use consistent section padding
- Alternate neutral backgrounds only when useful
- Keep content widths controlled for executive readability
- Avoid heavy decorative effects

## Cards
- Use cards for capabilities, industries, and partner principles
- Keep border radius modest
- Prefer subtle borders over strong shadows
- Keep card text short

## CTA Bands
- Use direct, executive language
- Include one primary action
- Avoid long paragraphs in CTA sections

## Forms
- Keep fields grouped logically
- Use dropdowns for organization type and area of interest
- Confirmation state should be calm and professional
- Connect form submissions to the client's preferred CRM or email workflow

## Accessibility
- Maintain strong color contrast
- Use clear focus states
- Label all form fields
- Use semantic heading order
- Avoid text embedded inside images

## Interactions
- Keep animation subtle
- Use hover states for cards and buttons
- Avoid scroll-heavy effects that distract from advisory positioning

## Build Notes
- Build desktop and mobile layouts in parallel
- Create shared symbols/components before filling all pages
- Keep page names and slugs simple
- Preserve the tone from the content files during design revisions
